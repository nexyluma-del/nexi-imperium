# Chief Web - Projekt Workflow

Status: intern  
Datenklasse: D1  
Risiko-Level: R1  

## Workflow Uebersicht

```text
Lead/Idee
  -> Briefing
  -> Angebot
  -> Nexi Approval
  -> Draft
  -> Review
  -> Finalisierung
  -> Launch Approval
  -> Launch
  -> Wartung anbieten
```

## Phase 1 - Lead / Anfrage

Input:

- Branche
- bestehende Website
- grobes Ziel
- moeglicher Preisrahmen

Output:

- Lead-Notiz
- Risiko-Klasse
- empfohlener Angebotstyp

Wichtig:

- Keine Kontaktaufnahme ohne Freigabe.
- Keine automatisierte Kaltakquise ohne Rechtscheck.

## Phase 2 - Briefing

Output:

- ausgefuelltes Kunden-Briefing
- fehlende Materialien
- offene Fragen

Stop:

- Kundendaten D2/D3 unklar
- Heilung/Finanzen/Recht mit Claims

## Phase 3 - Angebot

Output:

- Angebot in Markdown/PDF-Entwurf
- Preis
- Scope
- Nicht-Scope
- Zeitplan

Approval:

- Vor Versand immer Nexi-Go.

## Phase 4 - Draft

Output:

- lokale Demo
- Seitenstruktur
- erster Textstand
- mobile Ansicht

Qualitaetscheck:

- klare Headline
- sichtbarer CTA
- Leistungen verstaendlich
- keine leeren Versprechen
- keine rechtlich riskanten Aussagen

## Phase 5 - Review

Review-Fragen:

- Versteht man in 5 Sekunden, was angeboten wird?
- Ist der naechste Schritt klar?
- Ist die Seite mobil sauber?
- Fuehlt die Seite zur Branche passend?
- Gibt es fehlende Inhalte?

## Phase 6 - Launch

Nur mit Go:

- Hosting
- Domain
- DNS
- Formulare
- Tracking
- oeffentliche Website

## Phase 7 - Wartung

Immer anbieten:

- technische Kontrolle
- kleine Aenderungen
- monatlicher Kurzreport
- Weiterentwicklung
